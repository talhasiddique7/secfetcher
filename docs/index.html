secfetcher

Download SEC EDGAR filings by quarter or year using the official master index.

Install

Requires Python 3.10+.

pip install secfetcher

If your system Python is externally managed (PEP 668), use a virtual environment:

python -m venv .venv
. .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install secfetcher

Quickstart (Python API)

Defaults: data_dir="data", forms=["10-Q"]. Override as needed.

Download one quarter

from secfetcher import download_quarter

download_quarter(
    year=2024,
    quarter=3,
    forms=["10-Q", "10-K"],
    data_dir="data",
    file_types=[".xml", ".htm", ".html", ".pdf"],
)

Download a full year (all quarters)

from secfetcher import download_year

download_year(
    year=2024,
    forms=["8-K"],
    data_dir="data",
    file_types=[".htm", ".html"],
)

CLI

Same options from the command line:

secfetch quarter --year 2024 --quarter 3 --forms 10-Q 10-K --data-dir data --file-types .xml .htm .html .pdf
secfetch year --year 2024 --forms 8-K --data-dir data --file-types .htm .html

Module form:

python -m secfetch --help

Output layout

Filings are stored by year and quarter:

data/
  index/
    master/
      2024/
        QTR3/
          master.idx
  filings/
    2024/
      QTR3/
        10-Q/
          0000320193/
            0000320193-24-000069/
              <downloaded files...>
  _state/
    manifest.json

Resume: Running again for the same year/quarter skips already-downloaded filings. Use redownload=True (API) or --redownload (CLI) to clear and re-download.

Exceptions

  • SecFetchError — base exception
  • MissingUserAgentError — SEC user-agent not provided
  • RateLimitedError — SEC rate limiting hit
  • MasterIndexParseError — master index parse failure

Index cache

After a quarter completes with no errors, secfetch removes the cached master index for that quarter under data/index/master/<year>/QTR<q>/.
